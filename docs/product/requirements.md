# Requirements  

## Functional Requirements

- The user can manually create a financial transaction  
- The user can upload a CSV file containing transactions  
- The system must parse and store transactions from the uploaded CSV  
- The user can view a list of all transactions  
- The system calculates the user's balance based on transactions  
- The system groups transactions by category  
- The user can view a monthly financial summary  

## Non-Functional Requirements

- The system must persist user data reliably  
- The system should handle at least 1,000 transactions per user  
- The system should respond to requests within an acceptable time (< 1s)  
- The system should be simple and easy to use
